# Ado Visual Director

把一个选题、文章或创意简报，转成一套有审美判断的视觉方案：视觉命题、主角、视觉态度、页面结构、图片尺寸、逐页提示词和质量检查清单。用户直接要求“生图、生成图片、出图”时，在完成视觉方案后继续调用可用图片生成工具。

默认审美按领域选择：科技、AI、开发工具、模型、算力、Agent、产品架构类，优先使用**蓝色冷感科技编辑风**；医学、临床、AI医疗、诊断、随访、患者、医院类，优先使用**研究档案式医学视觉**。其他未明确领域默认使用研究档案式编辑设计。

蓝色冷感科技编辑风参考本次“Kimi K2.7 Code”视觉方向：冷蓝、钢蓝、蓝灰、真实开发工作台、IDE/终端/模型选择器、任务清单、评估材料、小字细节、工程秩序和精细材料层次。研究档案式医学视觉继续使用纸白、冷灰、理性蓝、病历/报告/影像/医生确认、责任链和审核痕迹。

支持两类信息密度：`detailed`（字多细节多，教程/操作手册感）和 `minimal`（字少留白多，重点突出）。用户没有指定时，小红书普通知识卡默认 `balanced`，小红书教程卡默认 `detailed`，PPT 默认 `minimal`。

## 核心工作流

1. 理解输入：确认主题、受众、平台、页数、图片尺寸、领域，以及这组图最想表达的一句话判断。
2. 选择平台预设或自定义尺寸。用户提到平台、比例、宽高或页数时，读取 `references/platform-presets.md`。
3. 确认审美风格：读取 `references/design-philosophy.md`。科技/AI/开发工具/模型/算力/Agent/产品架构类再读 `references/technology-cold-blue.md`；医学相关再读 `references/ai-medical.md`。
4. 确认内容形态和信息密度：
   - 用户说“小红书教程卡、教程式、操作手册、步骤、字多细节多、多点字、多点细节”时，读取 `references/xiaohongshu-tutorial.md`，使用 `xhs-tutorial` 形态和 `detailed` 密度，不做 30% 删除。
   - 用户说“PPT、演示页、幻灯片”时，读取 `references/information-density.md`；可选择 `detailed`（字多细节多）或 `minimal`（字少留白多）。
5. 写出视觉导演方案：
   - 视觉命题：这组图背后没有直接写出来的那句话。
   - 主角：读者第一眼应该看到什么。
   - 视觉态度：秩序、边界、责任、照护、怀疑、开放、克制、冲突、未完成感，或更准确的态度。
   - 视觉母题：只有在帮助理解时才使用；如果会束缚主题，就改用统一材料、色彩、观看距离和编辑节奏。
   - 否决清单：哪些默认 AI 设计元素绝对不出现。
6. 设计页面结构：每页只讲一个意思，标题短，主视觉清楚，有页码时要能读。
7. 根据 `references/prompt-templates.md` 生成提示词。
8. 做删除实验：默认做 30% 删除；如果用户明确要求“字多细节多、教程式、不用删除30%”，改做“去噪但不减信息”，只删除廉价装饰和重复无效内容。
9. 如果用户给出或需要 `visual_plan.json`，用脚本渲染提示词包：

```bash
python3 scripts/render_prompt_pack.py --input visual_plan.json --out-dir output
```

## 直接生图

当用户明确说“直接生图”“生成图片”“出图”“做成图片”“帮我把图生成出来”时，不要只输出提示词：

1. 先按核心工作流生成视觉方案和最终提示词。
2. 读取 `references/image-generation.md`。
3. 使用可用图片生成工具逐张生成图片；多页图文按页码顺序生成。
4. 交付时同时给出图片、使用的提示词、尺寸、页码和质量检查结果。
5. 如果当前运行环境没有图片生成工具，就明确说明无法直接出图，并输出可复制到生图工具的最终提示词包。

## 参考文件怎么读

- 通用审美和质量检查：读 `references/design-philosophy.md`。
- 科技、AI、开发工具、模型、算力、Agent、产品架构：读 `references/technology-cold-blue.md`，优先用蓝色冷感科技编辑风。
- 平台、比例、尺寸、页数：读 `references/platform-presets.md`。
- 小红书教程卡、操作手册、多字多细节：读 `references/xiaohongshu-tutorial.md`。
- PPT 信息密度、字多细节多或字少留白多：读 `references/information-density.md`。
- AI+医学、医疗、临床、诊断、病历、医生、患者、影像、随访：读 `references/ai-medical.md`，优先用研究档案式医学视觉。
- 输出最终图片提示词或 `visual_plan.json`：读 `references/prompt-templates.md`。
- 用户直接要求生图：读 `references/image-generation.md`。

## 默认输出

大多数任务输出三部分：

1. `visual_plan`：视觉命题、主角、视觉态度、母题/系统、风格系统、否决清单、删除实验。
2. `pages`：逐页标题、正文、视觉焦点、现实场景层、图解结构层、图片提示词。
3. `visual_plan.json`：用户需要机器可读文件或要运行脚本时再提供。
4. 用户要求直接生图时：额外交付生成图片和每张图对应的提示词。
5. 每次调用生图工具时，都要在交付目录保存或更新 `生图提示词-完整日志.md`，记录每一次提示词、用途、输出路径、采用/废弃状态和原因。

中文内容默认用中文。画面内文字必须短，优先适合手机屏幕和生图模型识别。

小红书教程卡例外：允许更多文字和步骤细节，但必须分成清楚模块、短句、表单、批注、速查卡，不要变成密密麻麻的长段落。

## 底线

- 不把这个 skill 做成普通模板库。每次都先判断“这张图到底在表达什么态度”。
- 避免机器人主角、AI 大脑、赛博城市、3D 图标堆、蓝紫渐变 SaaS 模板、PPT 卡片堆、随机 Logo、水印、满屏小字。
- 不模仿具体在世艺术家，也不照抄具体作品或构图。只吸收编辑设计、展览排布、档案视觉、杂志专题的通用方法。
- 医学内容不能编造疗效、准确率、诊断结论或治疗建议；不能出现可识别患者信息；不能暗示 AI 独立完成最终诊断。
