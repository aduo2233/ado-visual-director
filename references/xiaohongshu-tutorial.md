# 小红书知识教程式

当用户说“小红书教程卡、教程式知识卡片、操作手册、步骤、多点字、多点细节、不用删除30%”时读这个文件。

## 定位

`xhs-tutorial` 是小红书知识卡片的一种专门形态：不是轻量观点海报，而是可保存、可照着操作的教程卡。它允许更多文字和细节，但仍然要保持研究档案式的层级、真实材料和审美判断。

默认规格：

- 平台：`xiaohongshu`
- 尺寸：`1242x1660`
- 比例：`3:4`
- 默认页数：`8`
- 信息密度：`detailed`
- 删除策略：不做 30% 删除；改做“去噪但不减信息”

## 适用场景

- 工具安装和使用教程
- 工作流拆解
- Prompt / Skill / Agent 使用手册
- AI产品方法论步骤卡
- 公众号文章转小红书教程图
- 需要放入真实操作截图、生成结果或案例演示的图文

## 页面结构

8 页默认结构：

1. 封面：一句话说明这个教程解决什么问题。
2. 为什么：解释它不是普通模板/普通提示词。
3. 方法框架：调用前先问哪些问题。
4. 默认风格：说明审美系统和风格关键词。
5. 安装/准备：命令、路径、最小输入格式。
6. 演示 1：先输出视觉方案或提示词。
7. 演示 2：继续直接生图或生成结果。
8. 速查卡：复制用法、仓库链接、保存建议。

如果用户给了真实操作图，可以把 2-3 张图放在第 6/7 页：一页讲“先生成方案”，一页讲“再直接生图/看结果”。不要每一页都塞截图。

## 视觉风格

在默认研究档案式基础上增加教程感：

- 真实纸张、文件夹、剪贴板、安装手册、表单、命令行截图、浏览器/聊天窗口截图
- 分区清晰：标题区、步骤区、演示区、速查区、链接区
- 蓝灰结构线、页码、标签、小标题、编号步骤
- 红笔圈注、下划线、箭头、审核章、便签，用来指出重点
- 细节可以多，但必须像“资料被整理过”，不能像“画面被填满”

## 文字规则

允许多字，但必须模块化：

- 大标题：8-18 个字，表达判断。
- 副标题：一句说明这页的使用场景。
- 正文：用 3-6 个短要点，不写长段落。
- 操作命令：放进终端或代码纸条，尽量完整。
- 速查内容：用卡片/表单/清单呈现。
- 链接：可以放底部条，尽量清楚。

不要要求模型生成极小的密集文字。需要保真的命令、链接和正文，可在提示词里说“尽量准确呈现”，但生图后仍要检查。

## 删除策略

用户要求教程式或多字多细节时，不执行“30% 删除实验”。改用：

- 删除廉价装饰，不删除必要步骤。
- 删除重复图标，不删除关键命令。
- 删除空泛形容词，不删除操作条件。
- 删除无意义边框，不删除模块分区。
- 删除看不清的小字，改成更大的短句或表格。
