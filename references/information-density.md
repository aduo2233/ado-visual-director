# 信息密度模式

当用户要求“字多细节多、教程式、多点细节、不用删除30%、字少留白多、重点突出、PPT版本选择”等信息密度控制时读这个文件。

## 三种模式

### `minimal`

字少、留白多、重点突出。适合 PPT 演示页、封面、Hero 图、单张观点图。

规则：

- 标题 8-14 个字，最多两行。
- 辅助文字 1-2 句。
- 流程节点 3-5 个。
- 主视觉占 60%-75%。
- 大标题远距离可读。
- 删除 30% 视觉噪音。
- 避免密集表格、长段落、小字说明。

提示词追加：

```text
信息密度：minimal。字少、留白多、重点突出，适合投影或快速浏览。大标题远距离可读，辅助文字不超过两句，流程节点控制在 3-5 个，主视觉占据画面主要区域。执行 30% 删除实验，删除重复说明、装饰图标和密集小字。
```

### `balanced`

观点和说明平衡。适合普通小红书知识卡、文章配图、社媒图。

规则：

- 每页一个判断。
- 标题短，正文 2-4 条要点。
- 有真实材料层和图解层。
- 做轻度删除，保证手机端可读。

提示词追加：

```text
信息密度：balanced。标题清楚，正文保留 2-4 条要点，有真实材料层和图解结构层，手机端预览时仍能看懂。删除无用装饰，但保留必要解释。
```

### `detailed`

字多细节多。适合教程、操作手册、安装说明、流程拆解、案例演示。

规则：

- 允许更多文字，但必须模块化。
- 用清单、步骤、表单、命令行、便签、批注呈现。
- 可以放真实操作截图或生成结果演示。
- 不做 30% 删除；做去噪但不减信息。
- 仍要避免满屏不可读小字。

提示词追加：

```text
信息密度：detailed。字多细节多，像教程和操作手册。允许放入更多步骤、短句、命令、表单、批注和案例演示，但必须分区清楚、层级明确、手机端可读。不做 30% 删除，只删除廉价装饰和重复无效内容。
```

## PPT 密度选择

`ppt-slide` 默认使用 `minimal`，因为演示页通常需要远距离可读。

用户可以明确切换：

- “PPT，字少留白多，重点突出” -> `density=minimal`
- “PPT，字多细节多，像课程讲义/培训页” -> `density=detailed`
- “PPT，普通说明，不要太空也不要太满” -> `density=balanced`

PPT detailed 不是企业汇报 PPT。它仍要保留研究档案式和视觉主角，只是增加：

- 更完整的步骤说明
- 小型速查表
- 关键定义框
- 备注或批注
- 证据材料截图/表单

PPT minimal 则强调：

- 一个大判断
- 一个中心视觉主角
- 1 条辅助解释
- 3-5 个节点或标签
- 大面积留白

## visual_plan 字段

可在 `visual_plan.json` 中增加：

```json
{
  "content_mode": "xhs-tutorial",
  "density": "detailed"
}
```

或：

```json
{
  "platform": "ppt-slide",
  "density": "minimal"
}
```
